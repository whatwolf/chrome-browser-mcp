# Autorunway 持久化说明

## ✅ 持久化行为

### 数据持久化

**所有数据都会持久化保存**，包括：

1. **文件夹句柄** - 保存在 `chrome.storage.local`
2. **图片列表** - 保存在 `IndexedDB`
3. **处理进度** - 保存在 `IndexedDB`（每张图片的状态）

### 页面刷新行为

**刷新页面时：**

```
✅ 文件夹句柄保持不变
✅ 图片列表保持不变
✅ 处理进度保持不变
✅ 自动恢复到最后的状态
```

**初始化流程：**

```javascript
页面加载 → 检查文件夹句柄
    ↓
有句柄 → 检查数据库
    ↓
数据库为空 → 扫描文件夹
    ↓
数据库有数据 → 恢复进度 ✅
```

### 何时清除数据

**只有以下操作会清除数据：**

1. **点击"重新选择文件夹"** - 需要确认
2. **点击"清除所有数据"** - 需要确认

**不会清除数据的操作：**

- ✅ 刷新页面
- ✅ 关闭浏览器
- ✅ 重启电脑
- ✅ 点击"重置所有图片"（只重置状态，不清除列表）

## 📊 数据流

### 首次使用

```
用户点击"选择图片文件夹"
    ↓
选择文件夹并授权
    ↓
扫描文件夹 → 保存到 IndexedDB
    ↓
开始处理图片
    ↓
每处理一张更新状态
    ↓
[页面刷新]
    ↓
从 IndexedDB 恢复进度 ✅
```

### 重新选择文件夹

```
用户点击"重新选择文件夹"
    ↓
弹出确认对话框
    ↓
用户确认
    ↓
清除 IndexedDB 所有数据
    ↓
选择新文件夹
    ↓
扫描新文件夹 → 保存到新数据库
```

### 重置所有图片

```
用户点击"重置所有图片"
    ↓
将所有图片状态改为"pending"
    ↓
保留图片列表
    ↓
保留文件夹句柄
    ↓
可以重新处理所有图片
```

## 🔍 状态说明

### 图片状态

| 状态 | 说明 | 持久化 |
|------|------|--------|
| `pending` | 待处理 | ✅ 保存 |
| `processing` | 处理中 | ✅ 保存 |
| `completed` | 已完成 | ✅ 保存 |
| `failed` | 失败 | ✅ 保存 |

### 状态转换

```
pending → processing → completed
              ↓
           failed (可重置回 pending)
```

**位置：** 同 chrome.storage.local

## 🛡️ 数据安全

### 持久化保证

- ✅ **浏览器关闭** - 数据不会丢失
- ✅ **电脑重启** - 数据不会丢失
- ✅ **扩展更新** - 数据不会丢失
- ✅ **页面刷新** - 数据不会丢失

### 可能丢失数据的情况

- ❌ **手动清除浏览器数据** - 会清除所有扩展数据
- ❌ **卸载扩展** - 会清除扩展数据
- ❌ **点击"清除所有数据"** - 故意清除
- ❌ **点击"重新选择文件夹"** - 故意清除

### 备份建议

如果需要备份数据：

1. **导出图片列表**（未来功能）
2. **不要随意清除浏览器数据**
3. **定期使用"重置所有图片"而不是"清除所有数据"**

## 🔄 权限管理

### 权限持久化

```
首次授权 → 权限保存在浏览器
    ↓
后续使用 → 自动使用权限
    ↓
权限检查 → 验证是否有效
    ↓
权限过期 → 提示重新授权
```

### 权限过期

**可能的过期原因：**
- 浏览器更新
- 长时间未使用（几周）
- 手动撤销权限

**过期表现：**
```javascript
检查权限 → 返回 'denied' 或 'prompt'
    ↓
提示用户重新选择文件夹
    ↓
重新选择 → 重新授权
```

## 📝 最佳实践

### 日常使用

1. ✅ **首次选择文件夹后，无需再次选择**
2. ✅ **可以随时刷新页面，进度不会丢失**
3. ✅ **处理完一批图片后，点击"重置所有图片"重新处理**
4. ✅ **只在需要更换文件夹时才点击"重新选择文件夹"**

### 避免误操作

1. ⚠️ **"重新选择文件夹"会清除所有数据，需要确认**
2. ⚠️ **"清除所有数据"会永久删除所有数据，需要确认**
3. ✅ **"重置所有图片"只重置状态，不会删除数据**

### 故障排除

**问题：刷新页面后进度丢失**

**原因：** 可能是 IndexedDB 数据被清除
**解决：** 
1. 检查是否使用了"清除所有数据"
2. 检查是否清除了浏览器数据
3. 重新选择文件夹

**问题：提示权限已失效**

**原因：** 浏览器撤销了文件夹访问权限
**解决：** 点击"选择图片文件夹"重新授权

## 🎯 设计目标

### 持久化设计原则

1. **用户数据无价** - 不轻易清除
2. **进度自动保存** - 无需手动保存
3. **刷新不丢失** - 随时可以刷新页面
4. **明确才清除** - 需要用户确认才清除数据

### 用户体验

- ✅ **首次配置简单** - 选择一次文件夹即可
- ✅ **后续使用无缝** - 自动恢复进度
- ✅ **状态清晰可见** - 随时查看处理进度
- ✅ **操作安全可靠** - 危险操作需要确认
