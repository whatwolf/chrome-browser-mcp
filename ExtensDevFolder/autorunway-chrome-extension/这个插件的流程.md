# Autorunway 插件工作流程

## 1. 启动流程

```
用户打开 Runway 网站 → 人工登录 → 进入视频制作页面 → 插件自动加载
```

---

## 2. 初始化流程

1. 用户通过插件选择图片文件夹
2. 扫描文件夹中的所有图片
3. 将图片信息录入 IndexedDB 数据库，初始状态标记为 `0`（待处理）

---

## 3. 图片上传与生成循环

**执行模式：** 独立循环运行

### 流程步骤：

1. 从 IndexedDB 获取下一张状态为 `0` 的图片
2. 模拟上传 1 张图片到 Runway
3. 判断上传完成后，在 IndexedDB 中标记状态为 `1`（已上传）
4. 输入标准提示词（可自定义）
5. 判断生成按钮可用后，点击生成
6. 循环继续，处理下一张图片

**循环结束：** 所有图片处理完成后，弹窗提示用户

---

## 4. 页面变化监听与任务管理

**执行模式：** 独立持续运行（基于 MutationObserver）

### 4.1 新任务检测

- 监听页面 DOM 变化
- 检测新出现的每一个 Card 元素（视频生成任务）
- 将任务 ID 录入 IndexedDB 数据库
- 初始状态标记为 `0`（生成中）

### 4.2 任务完成检测

- 持续监听 Card 元素
- 通过检测操作按钮是否出现来判断任务是否完成
- 任务完成后，状态标记为 `1`（已完成）
- 启动视频类型判断模块

### 4.3 视频类型判断与处理

**判断逻辑：**

- **非 4K 视频：**
  - 点击操作按钮，生成 4K 视频
  - 将普通视频任务 ID 状态标记为 `9`（不再监控）

- **4K 视频：**
  - 获取下载链接
  - 将任务 ID 状态标记为 `2`（待下载）
  - 发送至下载模块

---

## 5. 下载模块

**执行模式：** 独立持续运行（轮询 IndexedDB）

### 流程步骤：

1. 轮询 IndexedDB 数据库，查找状态为 `2` 的任务
2. 获取下载链接
3. 下载视频文件到指定文件夹
4. 下载完成后，任务 ID 状态标记为 `3`（已完成）

---

## ⚠️ 待优化事项

### 并发控制需求

**问题描述：**
- 点击图片发送生成视频
- 普通视频放大为 4K 视频
- 以上两个操作共享最多 2 个运算并发

**当前限制：**
- 当运算并发已满时，"生成"按钮会变为 `disabled` 状态
- 此时也不应该可以点击普通视频 Card 的 4K 按钮

**优化方案：**
1. 实现全局并发计数器（最大值为 2）
2. 在点击"生成"或"4K"按钮前检查并发数
3. 并发数满时，禁用相关按钮或加入等待队列
4. 任务完成时释放并发名额

---

## 状态码说明

| 状态码 | 含义 | 说明 |
|:------:|------|------|
| `0` | 待处理/生成中 | 图片等待上传或视频正在生成 |
| `1` | 已上传/已完成 | 图片已上传或视频已生成 |
| `2` | 待下载 | 4K 视频已就绪，等待下载 |
| `3` | 已完成 | 视频已下载到本地 |
| `9` | 不再监控 | 普通视频已升级为 4K，无需继续监控 |
